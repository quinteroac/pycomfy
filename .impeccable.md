## Design Context

### Users

Developers and technical users who want to generate images, video, and audio from text prompts using a ComfyUI-powered backend. They interact with the interface in a desktop browser while iterating on creative prompts, adjusting generation parameters (steps, CFG, resolution), and downloading results. The primary job-to-be-done is: *type a prompt, tune parameters, get high-quality media output with minimal friction*.

### Brand Personality

**Parallax** — precise, powerful, effortless.

- **Precise**: every affordance is intentional; no decorative clutter.
- **Powerful**: the interface should feel like a capable, professional-grade tool.
- **Effortless**: generation feels fast and responsive; status is always clear.

Emotional goal: quiet confidence — the user feels in control and trusts the tool.

### Aesthetic Direction

- **Theme**: Dark mode only. Deep-space dark backgrounds (#0d0d17 → #13131f → #1a1a2e) with a single vibrant purple accent (#7c3aed, WCAG AA-compliant against white at ~5.25:1).
- **Typography**: Inter (system-ui fallback). 14–15 px body; 600 weight for labels and headings. Letter-spacing 0.04 em on the app title.
- **Spacing scale**: 8 / 10 / 12 / 16 / 20 / 24 px. Gaps never less than 8 px; section padding 20–24 px.
- **Border radius**: 12 px for containers and buttons; 10 px for action buttons.
- **Borders**: rgba(255,255,255,0.08) hairlines — present but not prominent.
- **Animations**: entrance (bubbleIn 0.25 s ease-out), progress (progressPulse shimmer), media reveal (mediaReveal fade+scale), loading spinner. All gated behind `@media (prefers-reduced-motion: no-preference)`.
- **Anti-references**: avoid light mode, gradients for gradient's sake, card shadows, heavy iconography, or rounded-pill buttons.

### Design Principles

1. **Clarity over decoration** — every element must earn its place. If it does not communicate state or guide action, remove it.
2. **Dark-first, accessible always** — maintain WCAG AA contrast at minimum; prefer AAA where feasible without compromising the dark aesthetic.
3. **Touch-target discipline** — all interactive elements are at least 44 × 44 px; focus-visible styles are mandatory on every focusable element.
4. **Motion is functional** — animations convey state (loading, arrival, progress) and are never purely decorative. Respect prefers-reduced-motion.
5. **Progressive disclosure** — default state is minimal; parameters and options appear only when relevant, avoiding cognitive overload.

### Tech Stack (for context)

- React 18 + TypeScript + Vite (frontend bundle)
- CSS Modules per component; global tokens in `App.css` (`:root` variables)
- Vitest + @testing-library/react for unit/component tests
- No CSS framework or component library — all styles are hand-crafted
