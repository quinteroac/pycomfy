# Audit Report — Iteration 000044, PRD Index 002

## Executive Summary

The implementation covers all five inference submission endpoints, job status polling, SSE streaming, job list, and the health endpoint. Core REST and streaming functionality is complete and functional. However, two structural requirements are unmet: FR-1 specifies a dedicated `server/gateway.py` APIRouter mounted in `server/main.py`, but all routes are inlined in `server/app.py`; FR-4 (CORS middleware) is completely absent. Additionally, a **critical bug** in `cancel_job` compares `row["status"] != "pending"` but the queue stores status as `"queued"`, making cancellation always return HTTP 409. FR-3 SSE polling is implemented at 500ms but is not configurable via `PARALLAX_SSE_POLL_MS`.

## Minor Observations

1. **Dead-code status mapping** — `_map_status` in `list_jobs` and `if status == "pending"` in `get_job_status` both map `"pending"` → `"queued"`, but the queue never stores `"pending"`. Suggests a mid-development status vocabulary change.
2. **Unsafe uv fallback** — `server/submit.py` uses `sys.executable` as fallback when `uv` is not on PATH. This may run the subprocess in the wrong Python environment. Should raise an explicit error instead.
3. **Duplicate `JobResult`** — Defined in both `server/jobs.py` and `server/schemas.py`. Should be consolidated.
4. **Unused `queue.cancel()` method** — The `cancel_job` endpoint calls `queue.update_status(job_id, "cancelled")` directly rather than the dedicated `queue.cancel()` method, which has proper guard logic. The dedicated method should be used.
5. **No tests** — No gateway endpoint tests exist (expected for prototype phase; must be addressed in refactor).

---

## Conclusions and Recommendations

The implementation is functionally solid for happy paths: inference submission, SSE streaming, status polling, and health checks all work as specified. The following issues must be resolved:

1. **[Critical]** Fix `cancel_job` status check: change `row["status"] != "pending"` → `row["status"] != "queued"` (or better, delegate to `queue.cancel()`).
2. **[High]** Add `CORSMiddleware` to the FastAPI app, configurable via `PARALLAX_CORS_ORIGINS` env var (FR-4).
3. **[High]** Make SSE poll interval configurable via `PARALLAX_SSE_POLL_MS` env var (FR-3).
4. **[Medium]** Refactor routes into `server/gateway.py` as an `APIRouter` and create `server/main.py` as the app entry point (FR-1, FR-5).
5. **[Low]** Clean up dead-code status mappings, consolidate `JobResult`, use `queue.cancel()` in the cancel endpoint.

## Refactor Plan

### Step 1 — Fix cancel bug (Critical)
In `server/app.py`, `cancel_job()`: replace the status guard to use `queue.cancel()` which handles the correct `"queued"` status check. Alternatively change the literal check from `"pending"` to `"queued"`.

```python
# Before (broken):
if row["status"] != "pending":
    raise HTTPException(...)

# After (fix option A — use queue.cancel()):
cancelled = await queue.cancel(job_id)
if not cancelled:
    raise HTTPException(status_code=409, detail="job is already running or completed")
return {"cancelled": True}
```

### Step 2 — Add CORS middleware (FR-4)
In `server/app.py`:
```python
import os
from fastapi.middleware.cors import CORSMiddleware

_cors_origins = os.environ.get("PARALLAX_CORS_ORIGINS", "*").split(",")
app.add_middleware(CORSMiddleware, allow_origins=_cors_origins, allow_methods=["*"], allow_headers=["*"])
```

### Step 3 — Make SSE poll interval configurable (FR-3)
```python
_SSE_POLL_S = float(os.environ.get("PARALLAX_SSE_POLL_MS", "500")) / 1000.0
# in event_generator:
await asyncio.sleep(_SSE_POLL_S)
```

### Step 4 — Restructure to gateway.py + main.py (FR-1, FR-5)
- Create `server/gateway.py` with all routes on an `APIRouter(prefix="")`.
- Create `server/main.py` that builds the FastAPI app, adds middleware, and includes the router.
- Keep `server/app.py` as a re-export shim or remove it.

### Step 5 — Cleanup
- Remove dead `_map_status` function and `if status == "pending"` branch.
- Consolidate `JobResult` into `server/schemas.py` and import from there in `server/jobs.py`.
- Fix unsafe `sys.executable` uv fallback.
